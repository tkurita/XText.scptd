Managements of AppleScript's text item delimiters

The heart of text handling of AppleScirpt is convesion between list and string/Unicode text changing gtext item delimiters. But code of changing AppleScript's text item delimiters is complicated and ugly. It is bother to type "AppleScript's text item delimiters". It's too long.

AppleScript's text item delimiters is a global variable of the instance of AppleScript runtime. Threfore AppleScript's text item delimiters usually is kept during the process is living. It may cause trouble that expecting the default value of AppleScript's text item deliminators. And AppleScript's text item delimiters should be reverted after the business.

XText can help managing AppleScript's text item delimiters by handlers store_delimiters() and restore_delimiters(). store_delimiters save current AppleScript's text item delimiters into a global variable "_pre_delims". restore_delimiters revert AppleScript's text item delimiters to the value saved with store_delimiters. Between store_delimiters() and restore_delimiters(), AppleScript's text item delimiters can be changed freely.

All of instance methods revert AppleScript's text item delimiters if it is changed during processing.

On the other hand, following class methods change AppleScript's text item delimiters and do not revert it automatically i.e., these methods should be called between store_delimiters() and restore_delimiters().

  • replace
  • split
  • join_list
  • formatted_text

A block of store_delimiters() and restore_delimiters() can be nested. The count of calling restore_delimiters() must not exceed the one of store_delimiters()
